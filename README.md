# LeftHandDevice

Raspberry Pi Pico 2 W (RP2350) をベースに開発した左手デバイスのファームウェアと、専用の設定用Windowsアプリケーション（WPF）からなるシステムです。

PC側のアプリケーションから複雑なマクロパターンを構築し、シリアル通信経由でマイコン本体のEEPROMに直接記憶させるアーキテクチャを採用しています。これにより、専用アプリがバックグラウンドで起動していなくてもどのPCでも同一の動作を行ことができます。


## ギャラリー

### PCアプリケーション

マクロ設定のメイン画面。ドラッグ＆ドロップによるステップの順番入れ替えに対応。

マウス動作の一括キャプチャ機能。画面上で実際にクリックした座標を自動で取得し、マクロとして記録する。

レジストリを走査し、インストール済みアプリケーション（.exe）を一覧表示。任意のアプリ起動マクロの登録を簡略化。

## 主な機能と特徴

### ハードウェア (Raspberry Pi Pico 2 W)

* **トリガー判定**: 「単発押し」「同時押し」「複数回押し(マルチタップ)」を判別。ミリ秒単位のデバウンス処理と判定猶予時間の設定により、意図しない操作（誤爆）を防ぐ。

* **連続実行（ループ）モード**: 物理スイッチの切り替えにより、長押しやタップを契機としてマクロを自動ループさせる「モードB」へ移行する。

* **状態フィードバック**: 設定同期時や連続マクロ実行時など、デバイスの現在の状態に合わせて各ボタンのLEDが明滅する。

* **EEPROMへの永続化**: PCアプリで設定した最大30個のマクロパターンをマイコン内のEEPROMに保存。一度設定すれば、他のPCに接続してもそのままの環境で使用可能。

### PC設定アプリケーション (Windows WPF / .NET)

* **Win32 APIを利用した画面キャプチャ**: OSの低レベルフック（`WH_MOUSE_LL`, `WH_KEYBOARD_LL`）を利用し、マウスの実際の軌跡やクリック座標、キーボード入力をそのままマクロとして記録する。手動での座標入力作業を排除。

* **自動アップデート機能**: GitHub APIをポーリングし、リポジトリに新しいリリースが存在する場合はアプリ起動時に通知を行う。

## 技術スタックとシステム構成

*↑ マイコン（C++）とWindowsアプリ（C#）間の連携システム構成図。*

* **デバイス側 (Firmware)**

  * **マイコン**: Raspberry Pi Pico 2 W (Pico W 互換)

  * **言語 / 環境**: C++ / Arduino IDE

  * **使用ライブラリ**: `Keyboard`, `MouseAbsolute`, `EEPROM`

* **PCアプリケーション (Client)**

  * **言語 / フレームワーク**: C# / WPF (.NET 10.0-windows)

  * **使用パッケージ**: `MaterialDesignThemes`, `Newtonsoft.Json`, `System.IO.Ports`

  * **OS連携**: P/InvokeによるWin32 APIの呼び出し（低レベルフック、レジストリ操作）

* **通信プロトコル**

  * シリアル通信 (115200bps) 。`ADD_PATTERN` や `SET_STEP` などの独自のテキストコマンドによるデータ同期。

## システム設計における技術的アプローチ

* **状態遷移と時間軸の非同期管理 (C++)**:
  同時押しやマルチタップなどの判定を、`delay` 関数によるブロッキング処理に依存せず、`millis()` を用いた状態マシンとして実装。メインループの実行周期を高く保ち、レスポンスの遅延を防いでいる。

* **ハードウェアとソフトウェアの責務の分離**:
  UIの構築やファイル操作、OSAPIへのアクセスなどの処理負荷の高い処理はC#（PC）側で担保し、キーボード/マウスのエミュレーションやミリ秒単位の物理ボタン判定はC++（マイコン）側で行う設計としている。

* **OSレイヤーへのアクセス (C#)**:
  ユーザー定義マクロ作成時の負担軽減を目的とし、マウスの絶対座標の取得にWin32 APIのグローバルフックを実装。

## 環境構築・利用方法

### 1. ファームウェアの書き込み

1. Arduino IDEを開き、ボードマネージャから `Raspberry Pi Pico/RP2040` をインストールします。
2. ボードの設定を `Raspberry Pi Pico W` または `Pico 2 W` に変更します。
3. `RaspberryPi/LeftHandDevice.ino` をマイコンに書き込みます。

### 2. PCアプリケーションの実行

PC側アプリケーションの利用方法は、用途に合わせて2種類から選択できます。

#### A. リリース版を使用する（推奨）

ソースコードのビルド環境がなくても、すぐにアプリケーションをご利用いただけます。

1. 本リポジトリの [Releases](../../releases) ページにアクセスします。
2. 最新バージョンの `LeftHandDeviceApp_vX.X.X.exe` をダウンロードします。
3. ダウンロードした実行ファイル（`.exe`）をダブルクリックして起動します。
4. 画面上部のコンボボックスから、Picoが接続されているCOMポートを選択し、「接続する」をクリックして同期を行います。


*↑ リリースページより、最新の実行ファイルをダウンロード可能です。*

#### B. ソースコードからビルドする（開発者向け）

ご自身で機能をカスタマイズしたり、コードを改変して使用する場合は以下の手順でビルドしてください。

1. [.NET 10.0 SDK](https://dotnet.microsoft.com/download) をインストールします。
2. Visual Studio 2022等で `WindowsApp/LeftHandDeviceApp.csproj` を開きます。
3. プロジェクトをビルドし、アプリケーションを起動します。
4. 画面上部のコンボボックスからCOMポートを選択し、接続を行います。
